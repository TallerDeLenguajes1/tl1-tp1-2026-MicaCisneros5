## Punto 2

### a) Archivo .gitignore

El archivo `.gitignore` se utiliza para indicarle a Git qué archivos o carpetas no deben ser incluidos en el repositorio.

Es conveniente incluirlo porque permite evitar subir archivos innecesarios, mantener el repositorio ordenado y proteger información sensible.

Se recomienda crearlo al inicio del proyecto, aunque también puede agregarse en cualquier momento.

Para configurarlo, se crea un archivo llamado `.gitignore` y se agregan reglas indicando qué archivos ignorar.

Por ejemplo, para ignorar un archivo llamado `ignorado.txt`, se debe escribir:

ignorado.txt
